Why we used Random Search for Public key e in RSA:

We use random search instead of a deterministic for-loop 
to find the public key e because it provides better
 cryptographic security and unpredictability. While a
  for-loop starting from a fixed value (3, 5, 7...)
   would quickly find a valid e, it makes the key generation
    process predictable, attackers could guess likely values 
    of e based on common patterns. Random selection ensures 
    that each key pair has a unique, unpredictable public key,
     enhancing security. Additionally, since the 
     probability of finding a coprime to Ï†(n) is very high, the expected number 
     of iterations is small (typically 1-3 attempts), making random 
     search both secure and efficient. This approach balances
      cryptographic strength with computational practicality,
       ensuring that RSA keys remain resistant to pattern-based attacks.
